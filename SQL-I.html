Data Types

The data type of a column defines what value the column can hold: integer, character, money, date and time, binary, and so on.

Char(size):
A FIXED length string. Size specifies the column length in characters (0–255).
Varchar(size):
A VARIABLE length string. Max size can go up to 65535 depending on row size.
Int(size):
Integer value. Size is display width, not the actual value range.
Date:
Stores date values in the format YYYY-MM-DD.
Boolean:
Stores TRUE or FALSE. Internally stored as 1 and 0.

Conditional Retrieval of Rows

To see the data of a table, we use the SELECT command. But by default, it gives all the rows of the table.
To retrieve specific rows, we use the WHERE clause combined with logical and comparison operators.

Types of operators in SQL with examples:

  • > (Greater than)
    SELECT * FROM students WHERE marks > 70;
    ➤ Shows students who scored more than 70 marks.

  • < (Less than)
    SELECT * FROM products WHERE price < 100;
    ➤ Fetches products that cost less than 100.

  • <= (Less than or equal to)
    SELECT * FROM employees WHERE age <= 30;
    ➤ Shows employees aged 30 or below.

  • >= (Greater than or equal to)
    SELECT * FROM orders WHERE quantity >= 5;
    ➤ Finds orders with 5 or more items.

  • = (Equal to)
    SELECT * FROM users WHERE city = 'Delhi';
    ➤ Retrieves users from Delhi.

  • AND (Both conditions must be true)
    SELECT * FROM students WHERE marks > 80 AND age < 18;
    ➤ Students who scored above 80 and are younger than 18.

  • OR (Either condition is true)
    SELECT * FROM books WHERE price < 100 OR author = 'John';
    ➤ Books under ₹100 or written by John.

  • NOT (Negates a condition)
    SELECT * FROM customers WHERE NOT city = 'Mumbai';
    ➤ Shows customers not from Mumbai.

  • IS (Checks for NULL)
    SELECT * FROM orders WHERE delivery_date IS NULL;
    ➤ Orders that have no delivery date yet.

  • IN (Matches any in a list)
    SELECT * FROM users WHERE country IN ('India', 'USA', 'UK');
    ➤ Users from India, USA, or UK.

  • IS NOT (Not NULL)
    SELECT * FROM tasks WHERE status IS NOT NULL;
    ➤ Tasks that have a status assigned.

  • NOT IN (Excludes a list)
    SELECT * FROM users WHERE id NOT IN (1, 2, 3);
    ➤ Users whose IDs are not 1, 2, or 3.

  • BETWEEN (Within a range)
    SELECT * FROM products WHERE price BETWEEN 100 AND 500;
    ➤ Products that cost between ₹100 and ₹500 (inclusive).

Matching a Pattern from a Table in SQL

To match a pattern in SQL, we use the LIKE operator.

It is often used with wildcards:

  • % → matches any number of characters (including zero)
  • _ → matches exactly one character

Ordering the Result of a Query & Aggregate Functions

To sort the results returned by a query, we use the ORDER BY clause.
You can sort in:

  • ASC – Ascending order (default)
  • DESC – Descending order

Example SQL Table:


            CREATE TABLE students (
                stId INT,
                name VARCHAR(50),
                class VARCHAR(20),
                marks INT
            );
            
            INSERT INTO students (stId, name, class, marks) VALUES
            (101, 'Riya', 'BCA-I', 88),
            (102, 'Ajay', 'BCA-II', 72),
            (103, 'Meena', 'BCA-I', 90),
            (104, 'Raj', 'BCA-II', 60),
            (105, 'Ayesha', 'BCA-I', 85);

            +------+--------+--------+--------+

            SELECT * FROM students ORDER BY name DESC;

Aggregate Functions are used to perform a calculation on a set of values and return a single value.
Common aggregate functions include:

  • COUNT() – Returns the number of rows
  • SUM() – Returns the total sum
  • AVG() – Returns the average value
  • MAX() – Returns the maximum value
  • MIN() – Returns the minimum value

Example 3: Count total students


            SELECT COUNT(*) FROM students;

Example 4: Find average marks


            SELECT AVG(marks) FROM students;

Example 5: Get the highest and lowest marks


            SELECT MAX(marks), MIN(marks) FROM students;

Example 6: Total sum of marks


            SELECT SUM(marks) FROM students;

Grouping the Result of a Query

In SQL, the GROUP BY clause is used to group rows that have the same values into summary rows.
It is often used with aggregate functions like COUNT(), SUM(), AVG(), MAX(), MIN().

Example SQL Table:


            +------+--------+--------+--------+
            | stId | name   | class  | marks  |
            +------+--------+--------+--------+
            | 101  | Riya   | BCA-I  |   88   |
            | 102  | Ajay   | BCA-II |   72   |
            | 103  | Meena  | BCA-I  |   90   |
            | 104  | Raj    | BCA-II |   60   |
            | 105  | Ayesha | BCA-I  |   85   |
            | 106  | Arjun  | BCA-II |   78   |
            +------+--------+--------+--------+

Example 1: Group students by class and count them


            SELECT class, COUNT(*) AS total_students
            FROM students
            GROUP BY class;

Output:


            +--------+----------------+
            | class  | total_students |
            +--------+----------------+
            | BCA-I  |       3        |
            | BCA-II |       3        |
            +--------+----------------+

Example 2: Find average marks of each class


            SELECT class, AVG(marks) AS avg_marks
            FROM students
            GROUP BY class;

Output:


            +--------+------------+
            | class  | avg_marks |
            +--------+------------+
            | BCA-I  |    87.66   |
            | BCA-II |    70.00   |
            +--------+------------+

Example 3: Show total marks by class


            SELECT class, SUM(marks) AS total_marks
            FROM students
            GROUP BY class;

Output:


            +--------+-------------+
            | class  | total_marks |
            +--------+-------------+
            | BCA-I  |     263     |
            | BCA-II |     210     |
            +--------+-------------+

Creation and Deletion of Views in SQL

A View in SQL is a virtual table based on the result of a SQL query. It can simplify complex queries and provide a layer of security.

To create a view:


                        CREATE VIEW student_view AS
                        SELECT name, marks
                        FROM students
                        WHERE marks > 75;

Sample Data in students Table:

                        SELECT * FROM student_view;
                        
                    +----+--------+-------+
                    | id | name   | marks |
                    +----+--------+-------+
                    | 1  | Raj    | 78    |
                    | 2  | Priya  | 92    |
                    | 3  | Ajay   | 65    |
                    | 4  | Meena  | 88    |
                    +----+--------+-------+

Output of student_view:


                        +--------+-------+
                        | name   | marks |
                        +--------+-------+
                        | Raj    | 78    |
                        | Priya  | 92    |
                        | Meena  | 88    |
                        +--------+-------+

This view filters only the students with marks greater than 75.

To delete (drop) a view:


            DROP VIEW student_view;

Output:


                    Query OK, 0 rows affected

The view student_view has been removed successfully.

GRANT, REVOKE, COMMIT, and ROLLBACK Commands in SQL

SQL provides control over access and transactions using the GRANT, REVOKE, COMMIT, and ROLLBACK commands. Below are the descriptions and examples.

GRANT Command

The GRANT command is used to give privileges to users.

Syntax:


            GRANT privilege_name ON table_name TO user_name;

Example:


            GRANT SELECT, INSERT ON students TO user1;

REVOKE Command

The REVOKE command is used to remove access rights given to a user.

Syntax:


            REVOKE privilege_name ON table_name FROM user_name;

Example:


            REVOKE INSERT ON students FROM user1;

COMMIT Command

The COMMIT command is used to permanently save any changes made by the current transaction.

Syntax:


            COMMIT;

Example:


            UPDATE students SET marks = 85 WHERE id = 1;
            COMMIT;

Before COMMIT:


            +----+--------+-------+
            | id | name   | marks |
            +----+--------+-------+
            | 1  | Rishi  | 78    |
            +----+--------+-------+

After COMMIT:


            +----+--------+-------+
            | id | name   | marks |
            +----+--------+-------+
            | 1  | Rishi  | 85    |
            +----+--------+-------+

ROLLBACK Command

The ROLLBACK command is used to undo transactions that have not been committed to the database.

Syntax:


            ROLLBACK;

Example:


            UPDATE students SET marks = 95 WHERE id = 2;
            ROLLBACK;

Before ROLLBACK:


            +----+--------+-------+
            | id | name   | marks |
            +----+--------+-------+
            | 2  | Ajay   | 88    |
            +----+--------+-------+

After ROLLBACK (No change committed):


            +----+--------+-------+
            | id | name   | marks |
            +----+--------+-------+
            | 2  | Ajay   | 88    |
            +----+--------+-------+

SQL Functions: Character, Date, and Group Functions

Character Functions

Character functions operate on character (string) data types and return character or numeric values.

1. UPPER() – Converts text to uppercase


            SELECT UPPER('sql tutorial');

Output:


            SQL TUTORIAL

2. LOWER() – Converts text to lowercase


            SELECT LOWER('SQL TUTORIAL');

Output:


            sql tutorial

3. LENGTH() – Returns the length of a string


            SELECT LENGTH('Database');

Output:


            8

Date Functions

Date functions work with date values and return formatted date or calculations.

1. CURRENT_DATE – Returns current system date


            SELECT CURRENT_DATE;

Output:


            2025-04-10

2. NOW() – Returns current date and time


            SELECT NOW();

Output:


            2025-04-10 15:42:00

3. DATEDIFF() – Finds difference between two dates


            SELECT DATEDIFF('2025-05-10', '2025-04-10');

Output:


            30

SQL Constraints

SQL constraints are used to specify rules for data in a table.

1. PRIMARY KEY

Ensures each row has a unique, non-null value.


                    CREATE TABLE std (
                        roll INT PRIMARY KEY,
                        name VARCHAR(20),
                        dob date,
                        city varchar(20),
                        class varchar(20),
                        age int
                    );

Output:


                        +-------+-------------+------+-----+---------+-------+

                        +-------+-------------+------+-----+---------+-------+

                        | name  | varchar(20) | NO   |     | NULL    |       |
                        | dob   | date        | YES  |     | NULL    |       |

                        | age   | int         | YES  |     | NULL    |       |

3. NULL and NOT NULL

`NOT NULL` ensures a column cannot have a NULL value.


            CREATE TABLE employee (
                emp_id INT,
                emp_name VARCHAR(50) NOT NULL
            );

4. UNIQUE

Ensures all values in a column are different.


            CREATE TABLE user (
                user_id INT,
                email VARCHAR(100) UNIQUE
            );

5. CHECK

Ensures the values in a column meet a specific condition.


            CREATE TABLE account (
                acc_id INT,
                balance INT CHECK (balance >= 1000)
            );

6. AUTO_INCREMENT

Automatically increments the value of a column with each new row.


            CREATE TABLE customer (
                cust_id INT AUTO_INCREMENT,
                cust_name VARCHAR(50),
                PRIMARY KEY (cust_id)
            );
